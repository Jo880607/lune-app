# Lune — CLAUDE.md

## 프로젝트 개요

**Lune** 은 "진짜 나를 알아가는 SNS"입니다.
AI가 나의 음성/텍스트 기록을 분석해 나를 해석해주고, 감정 기반으로 사람/AI와 연결되는 자기 탐색 플랫폼입니다.

- **타겟**: SNS는 지쳤는데 연결은 포기 못 한 25~35세
- **핵심 철학**: 보여주기 위한 기록이 아닌, 나를 알기 위한 기록
- **유저 한도**: 1,000명 (미접속 30일 시 교체)
- **수익 모델**: 아이템 판매 (하루 2,000원 한도, 구독 없음)

## 핵심 기능 목록

### 1. 기록 (Record)
- 음성 녹음 → STT → 텍스트 저장
- 텍스트 직접 입력
- 사진 없음 (의도적 제외)
- 올린 후 수정 불가 (24시간 내 삭제만 가능)

### 2. 나 (Me) — AI 분석
- 주 1회 (일요일 저녁) AI 분석 푸시 알림
- 감정 패턴, 자주 쓴 단어, 시간대 분석
- 기록이 쌓일수록 분석 정교해지는 구조
- Claude Haiku 사용 (비용 최적화)

#### 온보딩 첫날 예외 처리 (중요)
- 첫 기록 직후 즉시 AI 반응 제공 (주간 배치 예외)
- 복잡한 분석 아님 — "이 앱이 나를 듣고 있구나" 느낌만 주면 됨
- 예시 응답:
  1. "첫 기록을 남겼어요."
  2. "오늘 당신의 목소리엔 [감정]이 느껴졌어요."
  3. "매주 일요일, 쌓인 당신을 정리해드릴게요."
- 이후부터는 정규 주간 배치로 전환

### 3. 연결 (Connect)
- 사람 / AI / 상관없음 선택
- 하루 2회 왕복 제한 (펜팔 감성)
- AI 대화 시 항상 🤖 표시
- 7일 연속 AI 대화 시 사람 연결 제안

#### 연결 시스템 상세 설계
- **매칭**: 오늘 기록한 감정 태그 기반 매칭 (같은 감정끼리)
- **"상관없어요" 선택 시**: 랜덤 매칭
- **대화**: 하루 2회 왕복, 최소 2시간 간격
- **기록**: 대화 종료 후 전체 삭제. 단 내가 보낸 말은 내 기록 보관함에 저장
- **연장**: 상호 동의 시 1회만 연장 가능. 그 이상 불가
- **익명성**: 닉네임 + 픽셀 아바타만 노출

#### 대화 보관 정책
- 대화 종료 후 "이 대화를 보관할까요?" 팝업 표시
- 둘 다 동의 시 전체 대화 보관함에 저장
- 한 명이라도 거부 시 전체 삭제
- 24시간 내 응답 없으면 자동 삭제
- 대화 시작 전 "상대방이 이 대화를 보관하길 원할 수 있어요" 안내 표시
- 캡처 방지 기능 구현하지 않음

### 4. 아바타
- 자연물 픽셀 아바타 (별, 구름, 바람 등)
- 대화 상대에게만 보임 (피드 노출 없음)
- 유료 아이템 (하루 2,000원 한도) + 기록 달성 무료 아이템

### 5. 안전 정책
- 본인인증 필수 (PASS, NHN Cloud)
- AI 실시간 콘텐츠 필터 (음성/텍스트)
- 신고 3회 누적 시 자동 정지
- 영구 탈퇴 시 재가입 불가 (본인인증 기반)

## AI 분석 프롬프트 원칙

- 모델: Claude Haiku (claude-haiku-4-5-20251001)
- 호출 시점: 주 1회 배치 처리 (일요일 저녁)
- 분석 요소: 감정, 자주 쓴 단어, 시간대, 전주 대비 변화
- 톤: 감성적이고 따뜻하게. 판단하지 않음
- 금지: "당신은 ~해야 합니다" 같은 지시형 표현 사용 금지

```typescript
// AI 분석 호출 예시
const analysisPrompt = `
다음은 사용자의 이번 주 기록입니다.
판단하지 말고, 따뜻하고 감성적인 톤으로 분석해주세요.
지시형 표현은 사용하지 마세요.

기록: ${records}

아래 형식으로 응답해주세요:
- 이번 주 한 줄 요약
- 자주 등장한 감정 (최대 3개)
- 자주 등장한 단어 (최대 3개)
- 반복된 시간대
- 전주 대비 변화 (있을 경우)
`;
```

## 유저 한도 및 대기 시스템

- 최대 유저: 1,000명
- 미접속 퇴출: 30일 (14일 전 알림 발송)
- 대기자 명단: 순번 표시 + 친구 초대 시 순번 앞당김
- 퇴출 후 기록 데이터: 보관 (재입장 시 이어서 사용 가능)

## 단계별 확장 정책
- 1단계: 입장 1,000명 (시작)
- 2단계: 대기자 1,000명 돌파 시 → 입장 2,000명으로 확대
- 3단계: 대기자 다시 1,000명 돌파 시 → 입장 3,000명으로 확대
- 이후 동일하게 반복
- 확장 시 기존 대기자 순번대로 자동 입장 초대 문자 발송

## 온보딩 첫날 경험 (중요)

주간 AI 분석은 매주 일요일이지만, **첫 기록 직후에만 예외적으로 즉시 AI 반응**을 줍니다.

```
첫 기록 완료 직후 화면:

"첫 기록을 남겼어요."
"오늘 당신의 목소리엔 피로감이 느껴졌어요."
"매주 일요일, 쌓인 당신을 정리해드릴게요."
```

- 복잡한 분석 불필요. "이 앱이 나를 듣고 있구나" 느낌만 주면 됨
- 첫 기록 여부는 Firestore users 컬렉션 `firstRecordDone` 필드로 관리
- 한 번만 실행됨 (첫 기록 시에만)
